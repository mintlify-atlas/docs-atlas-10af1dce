---
title: "List Conversations"
api: "GET /v1/admin/conversations"
---

Retrieves a paginated list of all conversations in the system.

## Authentication

This endpoint requires admin authentication via the `x-admin-api-key` header.

<ParamField header="x-admin-api-key" type="string" required>
  Admin API key for authentication
</ParamField>

## Query Parameters

<ParamField query="limit" type="number" default={100}>
  Maximum number of conversations to return. Must be a positive integer up to 200.
</ParamField>

## Response

<ResponseField name="conversations" type="ConversationSummary[]">
  Array of conversation summaries
  
  <Expandable title="ConversationSummary properties">
    <ResponseField name="_id" type="string">
      Unique conversation identifier
    </ResponseField>
    
    <ResponseField name="_creationTime" type="number">
      Timestamp when the conversation was created (Convex internal)
    </ResponseField>
    
    <ResponseField name="sessionId" type="string">
      Session identifier associated with this conversation
    </ResponseField>
    
    <ResponseField name="createdAt" type="number">
      Timestamp when the conversation was created
    </ResponseField>
    
    <ResponseField name="updatedAt" type="number">
      Timestamp when the conversation was last updated
    </ResponseField>
    
    <ResponseField name="lastMessage" type="string">
      Preview of the most recent message in the conversation
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number">
  Total number of conversations in the system (before limit is applied)
</ResponseField>

## Example Request

```bash
curl -X GET "https://your-domain.com/v1/admin/conversations?limit=50" \
  -H "x-admin-api-key: your-admin-api-key"
```

{
